# Plan 10-01 — Polish, Deployment & Verification

## Goal
Add navigation, error boundaries, loading states, and 404 page. Restructure authenticated routes into `(app)` route group for shared layout.

## Tasks

### Task 1: Move authenticated route directories into `(app)` group
```bash
mkdir -p src/app/(app)
mv src/app/dashboard src/app/(app)/
mv src/app/transactions src/app/(app)/
mv src/app/budgets src/app/(app)/
mv src/app/analytics src/app/(app)/
mv src/app/settings src/app/(app)/
```

### Task 2: Create AppNav client component
**File**: `src/components/AppNav.tsx`

- `"use client"` directive
- `usePathname()` for active link
- Links: Dashboard, Transactions, Budgets, Analytics, Accounts
- Sign out button (form with server action)
- Mobile-responsive: wraps on small screens

### Task 3: Create `(app)` group layout
**File**: `src/app/(app)/layout.tsx`

- Server component, auth guard → redirect to /login
- Renders `<AppNav>` + content wrapper `max-w-6xl mx-auto`
- Passes `session.user` to AppNav for display

### Task 4: Create `(app)` error boundary
**File**: `src/app/(app)/error.tsx`

- `"use client"` required
- Props: `{ error: Error, reset: () => void }`
- Shows error message + retry button

### Task 5: Create `(app)` loading state
**File**: `src/app/(app)/loading.tsx`

- Simple animated skeleton or spinner

### Task 6: Create 404 page
**File**: `src/app/not-found.tsx`

- "Page not found" heading
- Link back to /dashboard

### Task 7: Remove redundant auth guards from individual pages
The `(app)/layout.tsx` now handles auth. Individual pages no longer need their own `auth()` + redirect calls. Remove them from:
- `(app)/dashboard/page.tsx`
- `(app)/transactions/page.tsx`
- `(app)/budgets/page.tsx`
- `(app)/analytics/page.tsx`
- `(app)/settings/accounts/page.tsx`

### Task 8: Add horizontal scroll wrapper to transactions table
In `src/app/(app)/transactions/_components/TransactionsClient.tsx`, wrap table in `<div className="overflow-x-auto">` for mobile.
